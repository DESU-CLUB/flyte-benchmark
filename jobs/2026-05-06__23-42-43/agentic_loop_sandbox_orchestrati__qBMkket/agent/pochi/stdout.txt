import dataclasses
from typing import Optional

env = flyte.TaskEnvironment(name='agent-env')

@dataclasses.dataclass
class StepResult:
    step: int

    result: Optional[float]

@env.task
async def add_numbers(a: float, b: float) -> float:
    return a + b

@flyte.trace
async def select_tool(instruction: str) -> str:
    return 'add_numbers'

@env.task
async def agent_loop(instructions: list[str]) -> list[StepResult]:
    results = []
    for i, inst in enumerate(instructions, 1):
        tool_name = await select_tool(inst)
        run = await flyte.run.aio(add_numbers, float(i), float(i))
        out = await run.outputs.aio()
        results.append(StepResult(step=i, tool=tool_name, result=out.o0, status='success'))
    return results

r = flyte.run(agent_loop, ['a', 'b'])
print(r.outputs())

## What was built

### Components

| Component | What it does |
|---|---|
| `env = flyte.TaskEnvironment(name="agent-env")` | Shared execution environment for all tasks |
| `@env.task add_numbers(a, b)` | Returns `a + b` |
| `@env.task multiply_numbers(a, b)` | Returns `a × b` |
| `@env.task divide_numbers(a, b)` | Returns `a ÷ b`, raises `ZeroDivisionError` on `b == 0` |
| `@flyte.trace async select_tool(instruction)` | Parses natural-language text with regex keyword patterns (divide → multiply → add, in priority order), extracts two numeric operands, and returns a tool-call descriptor dict — every call is recorded in the Flyte action trace |
| `@env.task async agent_loop(instructions)` | Top-level task: iterates instructions, awaits `select_tool`, dispatches via `await flyte.run.aio()`, collects typed `StepResult` dataclass instances |
| `main()` | Runs the loop locally, builds a JSON summary, writes to `/home/user/flyte_project/agent_output.json` |

### Key implementation details

- **`@flyte.trace` on an async function** — the `select_tool` coroutine must be `async` so the trace decorator uses its `wrapper_async` path. A sync traced function called inside an async task deadlocks Flyte's syncify thread.
- **`await flyte.run.aio()`** — sub-task invocations from inside an async task must use the `.aio()` variant; the synchronous `flyte.run()` deadlocks for the same reason.
- **`list[StepResult]` return type** — using a `@dataclasses.dataclass` (rather than `list[dict]` / `Any`) gives Flyte a concrete JSON schema for serialisation, avoiding the pickle fallback.
- **`RuntimeUserError` handling** — Flyte wraps exceptions raised inside a sub-task with `flyte.errors.RuntimeUserError`; the agent loop catches this to record the divide-by-zero failure cleanly.
