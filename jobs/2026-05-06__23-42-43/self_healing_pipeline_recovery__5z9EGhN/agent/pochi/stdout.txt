## How it works

1. **Environment definitions** — `primary_env` (8 CPU / 16 Gi) and `fallback_env` (2 CPU / 4 Gi) are declared using `flyte.TaskEnvironment` with `flyte.Resources`.

2. **`primary_training`** (`@primary_env.task`) — raises `RuntimeError("Insufficient memory for primary training")` when `len(data) > 10`; otherwise returns the complex-NN result dict.

3. **`fallback_training`** (`@fallback_env.task`) — always succeeds, returning the lightweight logistic-regression result.

4. **`self_healing_train`** (`@fallback_env.task`) — `try/except RuntimeError` wraps the `await primary_training(data)` call. On failure it logs a warning and transparently reroutes to `await fallback_training(data)`, adding `"recovered": True/False` to the result.

5. **`run_experiment`** (`@fallback_env.task`) — uses `asyncio.gather` to run both `self_healing_train` calls concurrently, demonstrating parallel self-healing.

6. **`__main__`** — drives the pipeline with `large_data = list(range(20))` (triggers fallback) and `small_data = list(range(5))` (succeeds on primary), then writes the combined result to `experiment_result.json`.
