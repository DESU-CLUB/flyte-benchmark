### API adaptation: `flyte.Cache`

The spec calls for `flyte.Cache(ttl=datetime.timedelta(hours=1), version="v1")`, but the installed Flyte 2.0 SDK's `Cache` class takes `(behavior, version_override=...)` — there is no `ttl` or `version` parameter. The equivalent call used is:

```python
flyte.Cache(behavior="override", version_override="v1")
```

`behavior="override"` means the cache key is pinned to `version_override="v1"`, which is semantically identical to the spec's intent (an explicit, stable version string that you bump to invalidate).
