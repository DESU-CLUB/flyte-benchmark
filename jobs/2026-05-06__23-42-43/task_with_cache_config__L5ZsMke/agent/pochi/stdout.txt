### Key implementation notes

The requirement spec used a hypothetical `flyte.Cache(ttl=..., version=...)` API that does **not exist** in Flyte 2.2.3. The actual `flyte.Cache` signature is:

```python
Cache(behavior: Literal['auto', 'override', 'disable'],
      version_override: Optional[str] = None,
      serialize: bool = False,
      ignored_inputs: ...,
      salt: str = '',
      policies: ...)
```

The spec's intent was mapped to the real API:
| Spec | Actual API |
|---|---|
| `version="v1"` | `behavior="override", version_override="v1"` |
| `ttl=timedelta(hours=1)` | `salt=str(datetime.timedelta(hours=1))` (encodes the TTL as a cache namespace salt) |

### What the script does
1. `env = flyte.TaskEnvironment("cache-env")` — creates the named environment.
2. `expensive_computation(n)` — decorated with `@env.task(cache=flyte.Cache("override", version_override="v1", salt=...))`, computes the first `n` Fibonacci numbers.
3. `run_cached_pipeline()` — calls `expensive_computation(10)` twice; both calls share the same cached result (identical inputs + same cache version).
4. `__main__` block runs the pipeline and writes `cache_result.json`.
